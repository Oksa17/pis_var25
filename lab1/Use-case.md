# Use-case: Генерация рецепта из имеющихся ингредиентов

**Первичный актор:** Голодный пользователь

**Цель:** Получить пошаговый рецепт приготовления блюда из тех ингредиентов, которые уже есть в холодильнике, без похода в магазин.

**Предусловия:**
- Пользователь авторизован в системе.
- Пользователь имеет хотя бы один ингредиент в списке "Мой холодильник" (или ввёл их вручную).
- Пользователь не превысил лимит запросов (менее 10 рецептов за последний час).

**Основной поток (Happy Path):**

1. Пользователь открывает раздел "Рецепто-генератор" в приложении.
2. Система отображает текущий список ингредиентов из холодильника (яйца, хлеб, молоко, лук).
3. Пользователь нажимает кнопку "Что приготовить из того, что есть?".
4. Система проверяет каждый ингредиент на просрочку (сверяет с датами годности).
5. Система отправляет запрос к AI-сервису на генерацию рецепта.
6. AI-сервис возвращает рецепт (название, список нужных ингредиентов, пошаговую инструкцию).
7. Система сохраняет сгенерированный рецепт в историю пользователя.
8. Система создаёт чек-лист готовки с таймерами на каждый шаг.
9. Система отображает пользователю рецепт с пошаговой инструкцией и кнопкой "Начать готовку".
10. Пользователь видит уведомление "Приятного аппетита! 🍳".
11. Use-case завершён.

**Постусловия:**
- Рецепт сохранён в истории пользователя.
- В очереди уведомлений находится задача на отправку push-уведомлений "Пора перевернуть блюдо" (для шагов с таймером).
- Чек-лист готовки доступен в кэше для быстрого доступа.

**Альтернативные потоки:**

**3a. Пользователь хочет добавить ингредиенты вручную**
- Пользователь нажимает "Добавить ингредиент".
- Пользователь вводит название и срок годности (опционально).
- Система сохраняет ингредиент в список "Мой холодильник".
- Возврат к шагу 3 основного потока.

**3b. У пользователя пустой холодильник**
- Пользователь нажимает "Что приготовить?" при пустом списке.
- Система отображает сообщение: "В холодильнике ничего нет, сходи в магазин 🛒".
- Use-case завершается неудачей (рецепт не генерируется).

**6a. AI-сервис не отвечает (таймаут)**
- Система выполняет повторную попытку (2 раза с задержкой 1 секунда).
- При повторном таймауте система переключается на **кэш топ-5 базовых рецептов** (гренки, омлет, бутерброд, жареный лук, салат).
- Система помечает рецепт как "source: fallback_cache".
- Система отображает пользователю сообщение: "AI сейчас занят, держи классический рецепт из нашего сборника".
- Возврат к шагу 7 основного потока (сохранение в историю).

**8a. Пользователь хочет заменить ингредиент**
- На этапе просмотра рецепта пользователь нажимает "Ой, нет этого ингредиента".
- Пользователь выбирает отсутствующий ингредиент (например, соль).
- Система отправляет запрос к AI-сервису на поиск замены.
- AI предлагает альтернативу (например, "соль → соевый соус").
- Система обновляет чек-лист с заменой.
- Возврат к шагу 9 основного потока.

**Исключительные ситуации:**

**4a. Проверка на просрочку обнаружена (исчерпана)** — *все ингредиенты просрочены?*
- **Условие:** В списке есть хотя бы один просроченный ингредиент (например, яйца с датой годности вчера).
- Система удаляет просроченный ингредиент из списка доступных.
- Система добавляет предупреждение: "Яйца просрочены, не рискуй! Они не будут использованы в рецепте".
- **Если после удаления просрочки остались ингредиенты:**
  - Система продолжает выполнение с шага 5 с оставшимися ингредиентами.
  - Пользователь видит красное предупреждение рядом с рецептом.
- **Если все ингредиенты просрочены:**
  - Система отображает сообщение: "Все продукты в холодильнике просрочены! Срочно проведи ревизию и сходи в магазин".
  - Use-case завершается неудачей.

**5a. Превышен лимит запросов (10 рецептов за час)**
- **Условие:** Пользователь уже сгенерировал 10 рецептов за последние 60 минут.
- Система НЕ отправляет запрос к AI-сервису.
- Система возвращает HTTP-статус 429 (Too Many Requests).
- Система отображает сообщение: "Ты сегодня уже нагенерировал 10 рецептов. Отдохни 15 минут, кухня остывает 🧊".
- Use-case завершается неудачей (рецепт не генерируется).

**6a. Валидация данных не пройдена (превышено максимальное количество ингредиентов)**
- **Условие:** Пользователь добавил более 20 ингредиентов вручную или в холодильнике >20 позиций.
- Система отображает сообщение об ошибке: "Максимальное количество ингредиентов для одного запроса — 20. Удали лишнее или разбей на два запроса".
- Система подсвечивает список ингредиентов красной рамкой.
- Возврат к шагу 3 основного потока для исправления.

**7a. Ошибка при сохранении рецепта в базу данных (нарушение целостности, таймаут)**
- Система откатывает транзакцию.
- Система логирует ошибку с уровнем ERROR.
- Система отображает сообщение: "Не удалось сохранить рецепт из-за технической ошибки. Попробуй позже или обратись в поддержку".
- Use-case завершается неудачей (рецепт не сохранён, хотя AI мог его сгенерировать).

**8a. Очередь уведомлений недоступна (RabbitMQ/Kafka не отвечает)**
- **Условие:** Рецепт успешно сгенерирован и сохранён в БД, но `rabbitTemplate.convertAndSend()` выбрасывает исключение.
- Система успешно сохраняет рецепт в БД (транзакция фиксируется).
- Система логирует ошибку: "Notification queue unavailable, saving to outbox".
- Система сохраняет задачу на отправку уведомлений в локальную таблицу `outbox` со статусом `PENDING`.
- Пользователь видит сообщение: "Рецепт опубликован. Напоминания о готовке придут, как только сервис станет доступен".
- Системный администратор получает алерт о падении очереди.
- Фоновый шедулер раз в 30 секунд пытается отправить отложенные уведомления.
- Use-case завершается успешно (с оговоркой).

**8b. Пользователь закрыл приложение во время готовки (таймер не сработал)**
- **Условие:** Пользователь начал готовку, но браузер/приложение было закрыто до получения уведомления "Пора перевернуть".
- Система хранит активные таймеры в Redis с TTL 30 минут.
- При возвращении пользователя система проверяет незавершённые чек-листы.
- Если время прошло, система отображает: "Твой таймер истёк, проверь блюдо! Напоминание: не отходи от плиты в следующий раз =)".

**9a. Ошибка при загрузке альтернативного ингредиента (замена не найдена)**
- **Условие:** Пользователь запросил замену для ингредиента, которого нет, но AI не нашёл аналог.
- Система отображает сообщение: "Замену для 'соль' не найдено. Может, добавим перец или лимонный сок?"
- Пользователь может выбрать предложенные варианты или отказаться от замены.
- При отказе — система предлагает удалить шаг с отсутствующим ингредиентом.
- Возврат к шагу 9 основного потока с обновлённым чек-листом.
