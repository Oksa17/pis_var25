### 2. Command Handlers

**Созданные обработчики:**

1. **GenerateRecipeHandler** - обработчик генерации рецепта из ингредиентов
   - Шаги обработки: получение пользователя → валидация ингредиентов → проверка лимита запросов → фильтрация просрочки → создание агрегата → сохранение рецепта → отправка уведомления → публикация событий
   - Возвращает: список доменных событий (List[DomainEvent])
   - Файл: `application/command/handlers/generate_recipe_handler.py`

2. **StartCookingHandler** - обработчик начала приготовления рецепта
   - Шаги обработки: получение рецепта → проверка прав пользователя → валидация длительности → создание сессии → сохранение сессии → отправка уведомления
   - Возвращает: список доменных событий (List[DomainEvent])
   - Файл: `application/command/handlers/start_cooking_handler.py`

3. **CompleteStepHandler** - обработчик завершения шага готовки
   - Шаги обработки: получение сессии → завершение шага (с проверкой инвариантов) → сохранение сессии → уведомление о завершении → проверка завершения всех шагов
   - Возвращает: список доменных событий (List[DomainEvent])
   - Файл: `application/command/handlers/complete_step_handler.py`

4. **RateRecipeHandler** - обработчик оценки рецепта
   - Шаги обработки: получение рецепта → валидация оценки (1-5) → проверка статуса рецепта (должен быть завершён) → добавление оценки → сохранение → уведомление
   - Возвращает: None
   - Файл: `application/command/handlers/rate_recipe_handler.py`

**Пример кода handler (GenerateRecipeHandler):**

```python
from src.domain.value_objects.ingredient import Ingredient
from src.domain.aggregates.cooking_session import CookingSession
from src.application.command.generate_recipe_command import GenerateRecipeCommand


class GenerateRecipeHandler:
    """Обработчик команды генерации рецепта"""

    def __init__(self, user_repository, recipe_repository, notification_service):
        self._user_repository = user_repository
        self._recipe_repository = recipe_repository
        self._notification_service = notification_service

    def handle(self, command: GenerateRecipeCommand):
        # 1. Получить пользователя
        user = self._user_repository.find_by_id(command.user_id)
        if not user:
            raise ValueError(f"Пользователь {command.user_id} не найден")

        # 2. Валидация: не пустой список ингредиентов
        if not command.ingredient_names:
            raise ValueError("Нельзя генерировать рецепт без ингредиентов")

        # 3. Валидация: лимит запросов (10 в час)
        if not user.can_generate_recipe():
            raise ValueError(f"Превышен лимит запросов. Не более 10 в час")

        # 4. Создать Value Objects и проверить просрочку
        ingredients = [Ingredient(name=name) for name in command.ingredient_names]
        fresh_ingredients = [i for i in ingredients if i.is_available()]

        if not fresh_ingredients:
            raise ValueError("Все ингредиенты просрочены!")

        # 5. Создать агрегат и сгенерировать рецепт
        session = CookingSession()
        recipe = session.generate_recipe(user, fresh_ingredients)

        # 6. Сохранить рецепт
        self._recipe_repository.save(recipe)

        # 7. Отправить уведомление
        self._notification_service.notify(
            user.id,
            f"Рецепт '{recipe.name}' сгенерирован! Приятного аппетита!"
        )

        # 8. Вернуть события
        return session.get_events()
```

### 3. Queries (Запросы)

**Созданные запросы:**

1. **GetRecipeByIdQuery** - запрос получения рецепта по идентификатору
   - Поля: `recipe_id: str`
   - Файл: `application/query/get_recipe_by_id_query.py`

2. **GetUserHistoryQuery** - запрос истории рецептов пользователя
   - Поля: `user_id: str`, `status_filter: Optional[str]`, `limit: int`, `offset: int`
   - Файл: `application/query/get_user_history_query.py`

3. **GetPopularRecipesQuery** - запрос популярных рецептов (по рейтингу)
   - Поля: `limit: int`, `min_rating: int`
   - Файл: `application/query/get_popular_recipes_query.py`

**Read DTOs:**

- **RecipeDTO** - упрощённая модель для чтения рецепта
   - Поля: `id`, `user_id`, `name`, `ingredients`, `steps`, `status`, `source`, `rating`, `created_at`
   - Файл: `application/query/dto/recipe_dto.py`

- **StepDTO** - упрощённая модель для чтения шага приготовления
   - Поля: `order`, `description`, `duration_minutes`, `is_completed`
   - Файл: `application/query/dto/step_dto.py`

**Пример кода (Query и DTO):**

```python
# get_user_history_query.py
from dataclasses import dataclass
from typing import Optional


@dataclass(frozen=True)
class GetUserHistoryQuery:
    """Запрос: получить историю рецептов пользователя"""
    user_id: str
    status_filter: Optional[str] = None  # "сгенерирован", "готовится", "завершён", "провален"
    limit: int = 20
    offset: int = 0
```

### 4. Query Handlers

**Созданные обработчики запросов:**

1. **GetRecipeByIdHandler** - обработчик получения рецепта по ID
   - Репозиторий: `RecipeRepository`
   - Возвращает: `RecipeDTO` или `None`
   - Файл: `application/query/handlers/get_recipe_by_id_handler.py`

2. **GetUserHistoryHandler** - обработчик истории рецептов пользователя
   - Репозиторий: `RecipeRepository`
   - Возвращает: `List[RecipeDTO]`
   - Файл: `application/query/handlers/get_user_history_handler.py`

3. **GetPopularRecipesHandler** - обработчик популярных рецептов
   - Репозиторий: `RecipeRepository`
   - Возвращает: `List[RecipeDTO]`
   - Файл: `application/query/handlers/get_popular_recipes_handler.py`

**Пример кода (GetUserHistoryHandler):**

```python
from src.application.query.get_user_history_query import GetUserHistoryQuery
from src.application.query.dto.recipe_dto import RecipeDTO


class GetUserHistoryHandler:
    """Обработчик запроса истории рецептов пользователя"""

    def __init__(self, recipe_repository):
        self._recipe_repository = recipe_repository

    def handle(self, query: GetUserHistoryQuery) -> List[RecipeDTO]:
        # 1. Получить все рецепты пользователя
        recipes = self._recipe_repository.find_by_user_id(query.user_id)

        # 2. Фильтрация по статусу (если указан)
        if query.status_filter:
            recipes = [r for r in recipes if r.status.value == query.status_filter]

        # 3. Сортировка по дате создания (новые сверху)
        recipes = sorted(recipes, key=lambda r: r.created_at, reverse=True)

        # 4. Пагинация
        paginated = recipes[query.offset:query.offset + query.limit]

        # 5. Конвертация в DTO
        return [
            RecipeDTO(
                id=r.id,
                user_id=r.user_id,
                name=r.name,
                ingredients=[i.name for i in r.ingredients],
                steps=[step.get_display_text() for step in r.steps],
                status=r.status.value,
                source=r.source,
                rating=r.rating,
                created_at=r.created_at
            )
            for r in paginated
        ]
```

### 5. Application Service (Фасад)

**Реализованный сервис:** `RecipeApplicationService`

**Методы:**

| Метод | Тип | Возвращает |
|-------|-----|------------|
| `generate_recipe(command)` | Command | List[DomainEvent] |
| `start_cooking(command)` | Command | List[DomainEvent] |
| `complete_step(command)` | Command | List[DomainEvent] |
| `rate_recipe(command)` | Command | None |
| `get_recipe_by_id(query)` | Query | RecipeDTO или None |
| `get_user_history(query)` | Query | List[RecipeDTO] |
| `get_popular_recipes(query)` | Query | List[RecipeDTO] |

**Пример кода:**

```python
from src.application.command.generate_recipe_command import GenerateRecipeCommand
from src.application.command.start_cooking_command import StartCookingCommand
from src.application.command.complete_step_command import CompleteStepCommand
from src.application.command.rate_recipe_command import RateRecipeCommand
from src.application.query.get_recipe_by_id_query import GetRecipeByIdQuery
from src.application.query.get_user_history_query import GetUserHistoryQuery
from src.application.query.get_popular_recipes_query import GetPopularRecipesQuery

from src.application.command.handlers.generate_recipe_handler import GenerateRecipeHandler
from src.application.command.handlers.start_cooking_handler import StartCookingHandler
from src.application.command.handlers.complete_step_handler import CompleteStepHandler
from src.application.command.handlers.rate_recipe_handler import RateRecipeHandler
from src.application.query.handlers.get_recipe_by_id_handler import GetRecipeByIdHandler
from src.application.query.handlers.get_user_history_handler import GetUserHistoryHandler
from src.application.query.handlers.get_popular_recipes_handler import GetPopularRecipesHandler


class RecipeApplicationService:
    """
    Application Service (Фасад) для Рецепто-генератора.
    Объединяет команды и запросы, скрывая детали обработчиков.
    """

    def __init__(
        self,
        user_repository,
        recipe_repository,
        session_repository,
        notification_service
    ):
        # Инициализация обработчиков команд
        self._generate_recipe_handler = GenerateRecipeHandler(
            user_repository, recipe_repository, notification_service
        )
        self._start_cooking_handler = StartCookingHandler(
            recipe_repository, session_repository, notification_service
        )
        self._complete_step_handler = CompleteStepHandler(
            session_repository, notification_service
        )
        self._rate_recipe_handler = RateRecipeHandler(
            recipe_repository, notification_service
        )

        # Инициализация обработчиков запросов
        self._get_recipe_by_id_handler = GetRecipeByIdHandler(recipe_repository)
        self._get_user_history_handler = GetUserHistoryHandler(recipe_repository)
        self._get_popular_recipes_handler = GetPopularRecipesHandler(recipe_repository)

    # ==================== КОМАНДЫ ====================

    def generate_recipe(self, user_id: str, ingredient_names: List[str]) -> List:
        """Сгенерировать рецепт из ингредиентов"""
        command = GenerateRecipeCommand(
            user_id=user_id,
            ingredient_names=ingredient_names
        )
        return self._generate_recipe_handler.handle(command)

    def start_cooking(self, recipe_id: str, user_id: str, estimated_minutes: int) -> List:
        """Начать приготовление рецепта"""
        command = StartCookingCommand(
            recipe_id=recipe_id,
            user_id=user_id,
            estimated_minutes=estimated_minutes
        )
        return self._start_cooking_handler.handle(command)

    def complete_step(self, session_id: str, step_number: int) -> List:
        """Завершить шаг приготовления"""
        command = CompleteStepCommand(
            session_id=session_id,
            step_number=step_number
        )
        return self._complete_step_handler.handle(command)

    def rate_recipe(self, recipe_id: str, user_id: str, rating: int) -> None:
        """Оценить рецепт"""
        command = RateRecipeCommand(
            recipe_id=recipe_id,
            user_id=user_id,
            rating=rating
        )
        return self._rate_recipe_handler.handle(command)

    # ==================== ЗАПРОСЫ ====================

    def get_recipe_by_id(self, recipe_id: str) -> RecipeDTO:
        """Получить рецепт по ID"""
        query = GetRecipeByIdQuery(recipe_id=recipe_id)
        return self._get_recipe_by_id_handler.handle(query)

    def get_user_history(
        self,
        user_id: str,
        status_filter: Optional[str] = None,
        limit: int = 20,
        offset: int = 0
    ) -> List[RecipeDTO]:
        """Получить историю рецептов пользователя"""
        query = GetUserHistoryQuery(
            user_id=user_id,
            status_filter=status_filter,
            limit=limit,
            offset=offset
        )
        return self._get_user_history_handler.handle(query)

    def get_popular_recipes(self, limit: int = 10, min_rating: int = 4) -> List[RecipeDTO]:
        """Получить популярные рецепты"""
        query = GetPopularRecipesQuery(limit=limit, min_rating=min_rating)
        return self._get_popular_recipes_handler.handle(query)
```

## Контрольные вопросы

**1. В чём разница между Command и Query?**

| Характеристика | Command (Команда) | Query (Запрос) |
|----------------|-------------------|----------------|
| **Назначение** | Изменяет состояние системы | Читает данные без изменений |
| **Возвращаемое значение** | ID или void | DTO или список DTO |
| **Побочные эффекты** | Да (сохранение, уведомления, события) | Нет |
| **Транзакционность** | Требуется | Не требуется |
| **Пример** | `GenerateRecipeCommand` | `GetRecipeByIdQuery` |

**2. Почему Command Handler возвращает только ID, а не весь объект?**

Команда должна быть максимально лёгкой и быстрой. Возврат всего объекта (например, `RecipeDTO`) смешивает команду и запрос, нарушая принцип CQRS. После выполнения команды клиент может сделать отдельный запрос для получения данных. Исключение — создание сущности, когда возвращается её идентификатор.

**Пример из моей системы:**

```python
# Команда возвращает только ID
command = GenerateRecipeCommand(user_id="user-1", ingredient_names=["хлеб"])
recipe_id = handler.handle(command)  # возвращает "abc-123"

# Отдельный запрос для получения данных
query = GetRecipeByIdQuery(recipe_id=recipe_id)
recipe = query_handler.handle(query)  # возвращает RecipeDTO
```

## Вывод

### Что получилось хорошо

Реализовано 4 Command Handlers (`GenerateRecipeHandler`, `StartCookingHandler`, `CompleteStepHandler`, `RateRecipeHandler`) и 3 Query Handlers (`GetRecipeByIdHandler`, `GetUserHistoryHandler`, `GetPopularRecipesHandler`). Все команды иммутабельны (`@dataclass(frozen=True)`), запросы не изменяют состояние системы. Фасад `RecipeApplicationService` объединяет все обработчики и делегирует вызовы.

### С какими трудностями столкнулись

Было сложно правильно разделить ответственность между Command Handler и Domain Model: что проверять в обработчике, а что в агрегате. Решили так: базовая валидация (существование сущностей, права доступа) — в Command Handler, бизнес-правила (лимиты, статусы, порядок шагов) — в Domain Model.

### Что узнали нового

Изучила паттерн CQRS: команды (изменение состояния) и запросы (чтение). Поняла роль DTO для изоляции доменного слоя. Узнала, почему Command Handler возвращает только ID, а не весь объект, и почему нельзя вызывать Query из Command Handler.
