### 3. E2E-тесты

**Сценарий:**
1. POST `/api/recipes/generate` → сгенерировать рецепт
2. POST `/api/recipes/{id}/start` → начать готовку
3. POST `/api/sessions/{id}/step/1` → завершить шаг 1
4. POST `/api/sessions/{id}/step/2` → завершить шаг 2
5. GET `/api/recipes/{id}` → проверить статус "завершён"
6. POST `/api/recipes/{id}/rate` → оценить рецепт

## Вывод

### Что получилось хорошо

Разработан полный комплект тестов для Рецепто-генератора: 25 юнит-тестов доменного слоя с покрытием 94%, 7 юнит-тестов прикладного слоя с моками зависимостей, 4 интеграционных теста с Testcontainers PostgreSQL, 3 E2E-теста API. Настроен CI/CD через GitHub Actions с автоматическим прогоном всех тестов при каждом коммите.

### С какими трудностями столкнулись

Настройка Testcontainers в CI окружении (требует Docker в раннере). Сложность мокирования цепочек вызовов в Command Handlers. Написание тестов для проверки порядка выполнения шагов готовки (инвариант "шаги по порядку").

### Что узнали нового

Научилась писать юнит-тесты для Value Objects, Entity и Aggregate Root. Освоила мокирование через `unittest.mock`. Научилась использовать Testcontainers для интеграционных тестов с реальной БД. Настроила CI/CD для автоматического тестирования. Поняла разницу между unit, integration и E2E тестами.
